# Campus Ledger ゲーム体験UI 設計

## 目的

簿記3級の学習効率と大学職員として成長する物語を維持しながら、毎日の案件に取り組みたくなるゲーム感を加える。明るく読みやすい教材領域と、濃紺を基調としたゲーム機風の操作領域を組み合わせる。

任天堂、Sonyおよび各社製品のロゴ、キャラクター、固有画面、効果音、アイコン配置は模倣しない。家庭用ゲーム機に共通する「明快な選択」「手応えのある操作」「達成の演出」だけを参考にした独自デザインとする。

教材、採点基準、復習間隔、模試、IndexedDB、バックアップ、PWA、完全オフライン動作は維持する。

## デザイン方針

- 濃紺と深い青をゲームUIの基調色にする。
- 教材カードは白または淡い青灰色とし、長時間でも読みやすくする。
- シアンを選択・進行、ライムを正解、オレンジを注意と誤答に使用する。
- 角を丸くしすぎず、斜めのアクセント、二重枠、発光線で軽快さを出す。
- ボタンには押下時の沈み込みと光の移動を加え、操作の手応えを示す。
- 外部フォント、外部画像、音声は追加せず、オフライン性能を維持する。
- `prefers-reduced-motion`では装飾アニメーションを停止する。

## 画面構成

### ゲームステータスヘッダー

既存の大学名、Day、通信状態、進捗を一つの濃紺パネルにまとめる。さらに現在の職員ランクと連続正解数を表示する。

- 左：大学章を抽象化した独自の文字マークとアプリ名
- 中央：`Day 1 / 14`、今日のテーマ、進捗ゲージ
- 右上：オンライン・オフライン状態
- ステータス列：職員ランク、連続正解数

連続正解数は炎などの一般的なゲーム記号を使わず、発光するチェーン型の独自記号と数字で表現する。数字だけでも意味が分かるよう「連続正解」と併記する。

### ミッションブリーフィング

先輩からの連絡を「MISSION BRIEFING」として表示する。初期状態は現在と同じ一行表示とし、展開時に会話全文を表示する。

- 案件カードの直前に配置する。
- 案件番号は `MISSION 1-1` の形式でも表示する。
- 英字は装飾的な補助表示とし、日本語を主情報にする。

### 案件カード

白い教材カードを濃紺の背景から浮かせ、ゲームのミッションパネルとして見せる。

- 上部に案件進捗、配点、書類名を表示する。
- 問題文は現在と同等以上の文字サイズとコントラストを保つ。
- 借方はシアン、貸方は青紫の識別線を使うが、文字見出しも必ず表示する。
- 入力欄は46px以上を維持する。
- 選択中の入力欄は発光枠で示す。

### 固定アクション

画面下部の提出ボタンをゲームの決定操作として強調する。

- 主ボタンはシアンから青のグラデーションと立体的な下辺を持つ。
- 押下時は1〜2px沈み、光沢が移動する。
- `仕訳を提出`、`次の案件へ`など、現在の操作名を維持する。
- 下部ナビゲーションと重ならず、360px幅でも操作できる。

### 下部ナビゲーション

3項目をゲーム機のメニュータブ風にする。選択中の項目は上側の発光線、背景色、文字色で区別する。

- 今日の業務
- 配属記録
- 保存

現在の1文字アイコンはCSSで描く独自の幾何学記号へ変更し、ラベルは必ず残す。

## 軽い成長要素

### 職員ランク

職員ランクは完了日数から算出し、追加の経験値は導入しない。

| 完了日数 | 職員ランク |
| --- | --- |
| 0〜2日 | 研修職員 |
| 3〜5日 | 経費担当 |
| 6〜9日 | 主任補佐 |
| 10〜14日 | 経理主任 |

ランクは保存済みの `completedDays` から毎回算出するため、新しい永続化項目を必要としない。ランク昇格時は次の画面表示で短い昇格バナーを出すが、操作を妨げるモーダルは使用しない。

### 連続正解

通常案件と復習問題を正解するたびに連続正解数を1増やす。誤答した時点で0へ戻す。誤答後に同じ問題へ正解した場合は1から再開する。

- 既に正解済みの問題を再表示しただけでは増やさない。
- 同じ回答を再送信しても重複加算しない。
- 模試は終了時に一括採点されるため連続正解の対象外とする。
- `currentStreak`と`bestStreak`を進捗へ保存する。
- 既存進捗と旧バックアップの読み込み時は、項目がなければ両方を0として補完する。

`LearnerProgress`のスキーマ番号は2へ更新し、IndexedDB読込とバックアップ読込に明示的なv1からv2への移行処理を追加する。回答、完了日、復習予定、模試結果はそのまま引き継ぐ。

## フィードバック演出

### 正解

- 案件カードの外周を短くライム色で発光させる。
- `案件クリア`と解説を表示する。
- 連続正解数を更新し、2以上なら小さなチェーン継続表示を出す。
- 紙吹雪や全画面演出は使わず、0.6秒以内の軽い演出にする。

### 誤答

- オレンジの枠と軽い横揺れを1回だけ表示する。
- `再確認が必要です`、ヒント、貸借不一致などの理由を表示する。
- 連続正解を0へ戻す。
- 失敗音や強い赤色の全画面表示は使わない。

### ランク昇格

- Day完了後、ヘッダー下に `RANK UP` と新しい職員ランクを表示する。
- 2.4秒後に静かに自動で消え、タップでも閉じられる。
- 昇格を見逃しても配属記録画面で現在ランクを確認できる。

## データとコンポーネント

### データ変更

- `LearnerProgress.schemaVersion`: `1`から`2`
- `LearnerProgress.currentStreak`: 現在の連続正解数
- `LearnerProgress.bestStreak`: 最高連続正解数
- v1進捗とv1バックアップをv2へ変換する移行関数

### UI単位

- `getStaffRank(completedDays)`: ランク名と次ランク条件を返す純粋関数
- `GameStatus`: Day、ランク、連続正解、通信状態を表示
- `MissionFeedback`: 正解、誤答、昇格の表示状態を担当
- 既存の`EntrySide`、問題カード、固定操作、下部ナビゲーションは構造を保ち、ゲーム用クラスと装飾を追加する

大規模な画面分割は行わず、成長計算と表示上の責務だけを分離する。

## アクセシビリティ

- ゲーム用装飾を除いて、主要文字はWCAG AA相当のコントラストを確保する。
- 色だけで正解、誤答、借方、貸方、選択状態を伝えない。
- 主要操作は44px以上、入力欄は46px以上を維持する。
- アニメーション後もフォーカス位置を移動しない。
- `aria-live`は正解、誤答、ランク昇格の短い通知だけに使用する。
- `prefers-reduced-motion: reduce`では発光、揺れ、光沢移動を無効化する。

## テスト

- 完了日数の境界で4種類の職員ランクが正しく算出される。
- 初回正解で連続正解が1になり、続けて正解すると増える。
- 誤答すると連続正解が0へ戻る。
- 同じ正解の再送信で連続正解が重複加算されない。
- 模試結果が連続正解へ影響しない。
- v1進捗とv1バックアップが回答履歴を維持したままv2へ移行される。
- 360px幅でランク、連続正解、問題、固定操作が横にはみ出さない。
- 正解、誤答、ランク昇格の表示がスクリーンリーダーへ通知される。
- モーション削減設定で装飾アニメーションが無効になる。
- サーバー停止後もService Workerから起動し、ランクと連続正解が復元される。

## 受け入れ基準

- 明るい教材領域と濃紺のゲームUIが一貫して見える。
- 特定企業やゲーム製品の模倣ではなく、Campus Ledger独自の外観になっている。
- 現在の職員ランクと連続正解数を今日の業務画面で確認できる。
- 通常案件と復習で正解すると連続記録が増え、誤答で0へ戻る。
- 完了日数に応じて職員ランクが変わる。
- 既存利用者の進捗とバックアップを失わない。
- 教材、採点、復習、模試、保存、オフライン利用が従来どおり動作する。
